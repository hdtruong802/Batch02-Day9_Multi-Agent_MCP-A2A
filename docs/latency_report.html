Tóm tắt

Hệ thống Stage 5 (5 service A2A: Customer → Law → Tax + Compliance) ban đầu không hoàn thành trong giới hạn timeout 900 giây. Sau các tối ưu kiến trúc và giảm số lần gọi LLM, cùng một câu hỏi hoàn tất trong 13.85 giây — cải thiện ít nhất ~65× so với ngưỡng timeout trước đó.

Đã cải thiện những gì & vì sao

  1. Keyword routing thay LLM check_routing (Law Agent)
    Mỗi request tiết kiệm ~5.94s (đo thực tế) và tránh thêm 1 round-trip LLM không cần thiết khi từ khóa đã đủ (tax, compliance, regulatory…). File: law_agent/graph.py — hàm route_to_subagents().
  2. Customer Agent: delegate trực tiếp, bỏ ReAct loop
    ReAct cần ít nhất 1–2 lần gọi LLM chỉ để quyết định gọi tool. Với entry-point luôn delegate sang Law, gọi thẳng A2A giảm latency và token. File: customer_agent/agent_executor.py.
  3. Tax & Compliance: 1 LLM call, bỏ create_react_agent không tool
    ReAct không có tool vẫn chạy agent loop — overhead vô ích. Chuyển sang StateGraph 1 node = 1 ainvoke. File: tax_agent/graph.py, compliance_agent/graph.py.
  4. Rút gọn system prompt (Law, Tax, Compliance, Customer)
    Giảm token input/output mỗi hop LLM; với free/slow API mỗi 100 token có thể thêm vài giây.
  5. Tax + Compliance chạy song song (LangGraph Send API)
    Thời gian ≈ max(tax, compliance) thay vì tổng tuần tự — đã có từ Stage 4, giữ nguyên trong Law Agent.
  6. Đổi model Gemini + tăng A2A_HTTP_TIMEOUT
    gemini-2.0-flash deprecated / quota 0; chuyển gemini-2.5-flash-lite. Timeout 900→1800s trong giai đoạn debug, sau tối ưu request xong <15s.
